# Medical Image Classification using AWS SageMaker

This project demonstrates an end-to-end Machine Learning pipeline on AWS SageMaker for classifying medical images. It covers data ingestion, hyperparameter tuning, model training with debugging/profiling, and deployment to a real-time endpoint.

## Project Setup Instructions
1. **Environment:** Use AWS SageMaker Studio or a Jupyter Notebook instance.
2. **Framework:** PyTorch 1.8.
3. **Data:** The dataset is fetched and uploaded to an S3 bucket. Ensure the S3 bucket URI is correctly updated in the notebook.
4. **Execution:** Run the cells in the notebook sequentially to perform training and deployment.

## File Descriptions
* **`Untitled.ipynb`**: The main driver notebook containing data processing, training job triggers, and deployment code.
* **`hpo.py`**: The entry-point script for training and hyperparameter tuning.
* **`Hyperparameters Report.pdf`**: Detailed report on model performance and deployment status.
* **`README.md`**: Project documentation and overview.

## Hyperparameter Tuning
I used SageMaker's Hyperparameter Tuner to find the optimal settings. 
* **Search Space:** * Learning Rate: Logarithmic range [0.001, 0.1]
    * Batch Size: [2, 4, 8]
* **Best Hyperparameters:** The most accurate model was achieved with a Learning Rate of **0.001** and a Batch Size of **2**.

## Debugging and Profiling
To monitor the training process, I integrated **SageMaker Debugger and Profiler**:
* **Debugging:** Used `SMDebug` rules to monitor the loss curve.
* **Profiling:** A detailed Profiler Report was generated to monitor system utilization and performance bottlenecks.

**Steps to Debug an Error:**
1. Review CloudWatch logs for Python exceptions.
2. Check the Debugger loss plot for convergence issues.
3. Verify S3 data permissions and paths.
